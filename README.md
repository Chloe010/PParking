# Pparking-基于飞浆库-JavaWeb的智能停车系统
### 项目目的
###### 本项目分析文档的目的是说明“小区物业车位管理系统”最终需要满足的条件和限制，设计的系统原型和数据库的字段实现，代码的逻辑层面对小区物业车位管理的功能实现做演示。

### 项目概述
#### 1、功能目标
###### 车位管理功能：包括添加车位信息、管理车位信息，管理车位信息模块可以对车主车位信息进行查询、编辑和删除；
###### 添加和修改用户功能：管理员寄予权限可以对用户进行添加与修改，同时用户通过注册可以对自身的资料信息进行编写和修改。
###### 识别用户身份功能：导出基于SpringBoot的车牌识别系统（飞浆库）进行对用户车牌号码的识别，并传输到数据库，最后在前端页面进行显示车牌号的具体信息。
###### 管理员增删改查功能：数据库数据系统显示所有用户信息后、通过后台索引，包括用户编号索引、用户车牌号索引、用户出入停车时间区间索引等进行对用户信息增删改查的管理功能；在固定车主停车管理中，可对车主的停车信息进行查询和删除；在临时车辆信息管理中，可对临时车主的停车信息进行查询和删除；在收费管理中，包括添加收费信息、管理收费信息。
###### 收益分析功能：通过获取车位管理系统在关于车位收益、小区管理的年月日收入比较、系统运营支出等具体情况寄予可视化分析，使用Tableau平台可视化分析后显示在前端页面。
###### 在临时卡信息管理：包括添加临时卡类型，管理临时卡类型。管理临时卡信息可以对车主的临时卡信息进行查询、编辑和删除，会对智能刷入卡进行处理，使用户进出更加方便。

#### 2、性能目标
###### 2.1精度
###### 停车取车过程中的时间以精确到“秒”为时间单位。
###### 进入时间和出场时间精确到秒
###### 调用接口飞浆库能正确扫描到车牌号
###### 2.2时间特性要求
###### 当前端用户信息和角色发生变化时，后台信息实时更新。
###### 用到的时间中，为了保证时间的可靠、公正，一律采用服务器的时间为	  标准。
###### 2.3故障处理要求
###### 鉴于车辆流动涉及的数据对于车主和管理员双方的重要性，数据严格有效的恢复机制；数据必须每天进行一次备份，由于本信息涉及信息量巨大，应以天为周期进行增量转储，以月为周期进行海量转储。
###### 2.4其它专门要求
###### 所有的查询、记录、修改等操作以按钮的形式给出，且名称一律取为“查询”，	“修改”，“删除”等易于理解的形式。

#### 4、系统设计
##### 4.1 划分模块
###### ①、管理员模块：
###### 包含查看此时剩余车位和车位位置情况、添加和删除角色和用户、添加和管理小区所有车位信息、识别用户身份对用户的增删改查进行操作、固定车主的停车管理模块和临时车辆的停车管理模块、分析停车场收益的管理模块、系统登录和改密操作。
###### ②、用户模块：
###### 查看自己的车辆进出停车场的时间表、对于用户的个人信息页面、修改个人的账号密码。
###### ③、值班人员模块：
###### 值班人员信息管理、车位剩余量查看、用户管理、车主出场信息等、修改值班人员的账号密码。
